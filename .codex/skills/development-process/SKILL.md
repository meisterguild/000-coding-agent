# Development Process

## Overview
プロジェクトを進めるための基本フローを定義する。
各ステップを順番に実行すること。
ただし、次のステップに進むには成果物に対する指示者の承認が絶対に必要。
成果物が作成出来たら次のステップに進まず、指示者に成果物を提出し、承認を得ること。

## Rules

- 各ステップが完了したら、必ずその成果物を提示し、「レビュー依頼」を出して作業を停止すること。
- レビューで「承認」と明示されるまで、次のステップに進んではならない。
- 承認がない状態で次工程に進むことは禁止。
- 自律的に最後まで進めないこと。
- 必ず人間の確認待ち状態で止まること。

## Steps
1. 要件定義
   - 目的、期待成果、制約を明確化する。
   - 成果物：docs/requirements.md
2. 設計
   - 要件に基づいてシステムの構造や仕様を設計する。
   - 成果物：docs/design.md
3. タスク分割
   - 設計を実行可能な単位に分解し、優先順位を決める。
   - 成果物：docs/tasks.md
4. 実装
   - タスクに沿って実装を行い、必要に応じて中間確認を行う。
   - 成果物：src/
5. 動作確認
   - 要件を満たしているか検証し、問題があれば修正する。
   - 成果物：docs/test_results.md
