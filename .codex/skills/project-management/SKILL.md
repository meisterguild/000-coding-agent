# Project Management

## Overview
プロジェクトを進めるための基本フローを定義する。
以下の各ステップで成果物のレビュー依頼を指示者に出してください。

## Steps
1. 要件定義
   - 目的、期待成果、制約を明確化する。
   - 成果物：docs/requirements.md
2. タスク分割
   - 要件を実行可能な単位に分解し、優先順位を決める。
   - 成果物：docs/tasks.md
3. 実装
   - タスクに沿って実装を行い、必要に応じて中間確認を行う。
   - 成果物：src/
4. 動作確認
   - 要件を満たしているか検証し、問題があれば修正する。
   - 成果物：docs/test_results.md
