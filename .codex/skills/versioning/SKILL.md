# Versioning

## Overview
ファイルの変更に関するルール。変更や追加をそのまま通すのではなく、どんな変更であってもレビュー依頼を出し、それがOKだった場合にgit commitすることでファイル変更履歴を残すルール。

## Prerequisites
.gitディレクトリがプロジェクトルートになければ、git initを実行する。

## Examples
以下は file-management スキルを使った典型的なワークフローを 4 つのケースに分けて示します。共通手順は「変更 → ステージング → コードレビュー → コミット」です。コミットメッセージはタスクタイトルがあればそれを、無い場合は日本語で簡潔に記載してください。

1. ファイルを追加した場合
   - 新規ファイルを作成し、git add でステージ。
   - コードレビューをリクエストし、承認後 git commit -m "タイトル" でコミット。

2. ファイルを削除した場合
   - git rm で削除しステージ。
   - 同様にレビュー→コミット。

3. ファイルを変更した場合
   - 変更後 git add でステージ。
   - レビュー→コミット。

4. ファイルを追加・削除・変更した場合（複数操作）
   - 追加、削除、変更それぞれを git add / git rm でステージ。
   - すべてステージ後にレビュー→コミット。
